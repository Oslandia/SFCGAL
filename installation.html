Installation

Source code

SFCGAL source code is available on the corresponding github repository.

The master branch is the development branch and has a tag for each release version.

Source code of the 1.0 release can be found with the tag v1.0. You can direclty download a zip or tarball.

Requirements

  • A C++ compiler, see the development section for supported platforms and compilers,
  • CMake 2.8.6+
  • CGAL 4.1 or higher
  • Boost 1.46 or higher
  • MPFR (CGAL Epeck) 2.2.1 or higher
  • GMP (for CGAL Epeck) 4.2 or higher

Optional dependencies (viewer and 3D format export)

  • OpenSceneGraph (version >= 3.1)
  • Qt4

    • Compilation

    The compilation process is based on CMake. The classic way of building is to first create a building directory and use a CMake client to configure the build. For instance using ccmake, the curses-based frontend for cmake (you can also use cmake-gui):

    mkdir build
cd build
ccmake ..

    Default building options should work out-of-the-box on a recent Linux distribution. You may want to fine tune the compilation process.

    CMake build options
    • CMAKE_INSTALL_PREFIX (default: /usr/local/) Specifies where SFCGAL will be installed when make install is invoked
    • CMAKE_BUILD_TYPE (default: Release) is used to switch between a Release build, specially crafted for speed efficiency and a Debug build, specifically crafted for development. The Debug build will also set CGAL to be used in Debug mode. In this case, lots of assertions are kept present in the code and it results in a very slow code.
    SFCGAL specific build options
    • SFCGAL_CHECK_VALIDITY (default: ON) If set to ON, geometries will be testes for validity before actually calling algorithms. Turn to OFF to disable the validity check. Refer to the reference manual.
    • SFCGAL_BUILD_BENCH (default: OFF) Turn to ON to build benchmark tests
    • SFCGAL_BUILD_EXAMPLES (default: OFF) Turn to ON to build examples
    • SFCGAL_BUILD_OSG (default: OFF) Turn to ON to enable the use of OpenSceneGraph. It should be turned to ON to use the viewer
    • SFCGAL_BUILD_REGRESS_TESTS (default: OFF) Turn to ON to build regression tests
    • SFCGAL_BUILD_UNIT_TESTS (default: OFF) Turn to ON to build unit tests
    • SFCGAL_BUILD_VIEWER (default: OFF) Turn to ON to build the viewer
    Miscellaneous options
    • SFCGAL_BUILD_FAILED_ON_WARNING (default: OFF) Turn to ON to convert warnings into errors
    • SFCGAL_BUILD_WITH_GPROF (default: OFF) Turn to ON to build with GNU gprof
    • SFCGAL_USE_STATIC_LIBS (default: OFF) If set to ON, compile as a static library. Do not change this setting unless you know what your are doing
    Dependencies location (environment variables)
    GMP_DIR : GMP location
MPFR_DIR : MPFR location
CGAL_DIR : CGAL location
BOOST_ROOT : Boost location

    One everything is properly configured, type make to compile (do not forget a -j X if you have X processors to dedicate to the building). You can install the library with make install as usual.

    CGAL uses a lot of templated constructions. The whole building process may then take a while.

    Troubleshouting

    Windows/MinGW

    • MinGW fail to build boost thread for version 1.46.1 (fixed in boost 1.47.0, please upgrade your boost version)
    • Use the same cmake option for boost with CGAL and SFCGAL (particularly Boost_USE_STATIC_LIBS=OFF)
    • Missing boost definitions in CGAL-4.1 to use static libraries (dllexport/dllimport)

    Patch CGAL-4.1/CMakeLists.txt with the following lines to build with boost static libraries (Boost_USE_STATIC_LIBS=ON).

    option( Boost_USE_STATIC_LIBS "boost use dynamic libraries" OFF )
if( Boost_USE_STATIC_LIBS )
  message( STATUS "Boost_USE_STATIC_LIBS=ON" )
    add_definitions( "-DBOOST_THREAD_USE_LIB" )
else()
    message( STATUS "Boost_USE_STATIC_LIBS=OFF" )
    add_definitions( "-DBOOST_ALL_DYN_LINK" )
endif()

    Installation with PostGIS

    On a recent PostGIS source tree (2.1beta for instance), an additional option for the configure script called --with-sfcgal is available. Set it to wherever your SFCGAL installation is. If you used the default directory /usr/local and if the bin sub-directory is present in your PATH, then SFCGAL should be detected without any aditionnal options.
